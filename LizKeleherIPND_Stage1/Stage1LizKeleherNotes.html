Work Session 2 Notes

It's all a BOX

It's helpful to realize everything on the web is a box. Now, when I'm thinking about how I'm going to write my code, I can visualize what boxes go where and how to divide things.

WOW

HTML and CSS work together for structure and style. You can use tags that add specific colors or resize the font.

Everything on the web page is a box. Your HTML determines the structure of the boxes and you can change the style of the boxes with CSS.

First, you create a shell of the page you are trying to create by thinking in terms of how many boxes and where they will go. Then, you create reminders for yourself about what you will put in each with "classes" like title, image, etc...

WONDER

I'm still challenged to visualize where the different elements of CSS come in to work together. I understand they are both needed to create structure and style, but I want to learn more about how they are partners to support one another and how I'll keep the two that way in my mind as I write my code. In codepen I see that each line of HTML has a corresponding line of CSS- is this how I add that in? I put the CSS in the same line?

Avoid Repetition

Always think about minimizing the amount of code you have to write. You can do this by organizing it and applying the CSS rules in order to bucket things together. This will also help if you work on it again or if you let other people work with your code.
